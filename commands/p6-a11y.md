# /p6-a11y – Accessibility Check

Systematically checks the application for accessibility. Each A11y dimension is a separate sub-skill.

## Argument: $ARGUMENTS = [page/component]

If provided: Check the specified page/component.
If not provided: Check all critical screens.

## Flow

### 0. Ensure Sub-Index Exists
Make sure `docs/quality/A11Y.md` exists as a **sub-index** with the standard header (Status / Last Updated / Key Decisions / Open Risks / Detail Files / Gate Notes — see `~/.claude/docs/PROJECT_PHASES.md`). If missing, create it with empty placeholders. The three `p6-a11y-*` sub-skills below each refresh their own row in this sub-index's **Detail Files** table.

### 1. Visual Accessibility
`/p6-a11y-visual $ARGUMENTS` – Contrast, colours, dark mode, red-green colour blindness.

### 2. Keyboard Navigation
`/p6-a11y-keyboard $ARGUMENTS` – Tab order, focus, Enter/Space, Escape.

### 3. Screen Reader Compatibility
`/p6-a11y-screenreader $ARGUMENTS` – ARIA, semantics, alt texts, live regions.

### 4. Roll Up Sub-Index to Phase Index
After all `p6-a11y-*` sub-skills have run, summarise the A11Y sub-index into the phase index `docs/quality/QA.md`:
- Add an entry in the phase-index **Detail Files** table for `[A11Y.md](A11Y.md)` (the sub-index itself), status `complete` once all three sub-skill rows are `complete` (or `needs-rework` if blocking findings remain).
- Lift any unmitigated A11y blocker into the phase-index **Open Risks**.

## Notes
- All three sub-skills can run in parallel (no dependencies)
- Detail files: `a11y_visual.md`, `a11y_keyboard.md`, `a11y_screenreader.md` under the `A11Y.md` sub-index
- WCAG 2.1 AA as minimum standard
- Always account for red-green colour blindness

### Handover Epilogue
Update `docs/HANDOVER.md`:
- What was created/changed
- Open points
- Next Steps (according to `~/.claude/docs/NEXT_STEPS_REFERENCE.md`)

Recommend 1-3 useful next commands to the user:
1. Read `docs/HANDOVER.md` for the current project status
2. Consult `~/.claude/docs/NEXT_STEPS_REFERENCE.md` for allowed transitions
3. Only suggest commands that match the current phase/sub-skill status
4. If the current phase appears complete: recommend the gate
